# Crypto LawMap — Vault

This vault is auto-generated by the pipeline in `../src/`. Every `.md` note is a legal norm; the YAML frontmatter carries its state; the `[[wikilinks]]` form the graph.

## How to read

| Field | Meaning |
|---|---|
| `id` | Unique identifier: `{COUNTRY}-{SLUG}-{YEAR}` (e.g. `BR-LAW14478-2022`) |
| `country` | ISO alpha-2 (`INTL` for FATF, FSB, BIS, IOSCO, …) |
| `type` | `statute` (law) · `regulation` · `guidance` · `case_law` (binding precedent) |
| `status` | `discovered` → `verified` → `scraped` → `analyzed` (or `quarantine`) |
| `discovered_via` | `seed` (initial discovery) · `citation` (cited by another norm) · `semantic` (suggested by similarity) |
| `confidence` | 0–1; high = primary source (gov/regulator) + live URL + title match |
| `source_authority` | `primary` (gov/regulator) · `secondary` (legal database) · `tertiary` (media) |
| `cycle` | Loop iteration in which the note was created |
| `title` | English display title (auto-translated when source is not English) |
| `title_original` | Title in the source language — for legal verification |
| `references` | List of `[[wikilinks]]` to other norms |
| `ref_types` | Per ref: `citation` (explicit) or `semantic` (suggestion) |

## Folder structure

```
BR/    US/    SG/    ...      → one folder per country, with national norms
INTL/                         → supranational norms (FATF, BIS, FSB, …)
_MOC/                         → Maps of Content (per-country indexes, auto-generated)
_meta/                        → run-log.md, budget-tracker.json
_quarantine/                  → low-confidence nodes (EXCLUDED from the main graph)
```

## Language policy

The vault is **English-first**. Frontmatter `title`, note bodies, MOCs, and dashboards are all in English. The original-language title is preserved in `title_original` and the original body text is available in a collapsed `<details>` section at the bottom of each scraped note, for legal-verification purposes. Source URLs (`source_url`) always point to the authoritative original-language text.

## Manual editing

You **can** edit any note by hand (annotations, marginalia, translation corrections). The pipeline does not overwrite the body of a note already at `status: analyzed` — it only performs non-destructive merges of new `references`. Avoid editing frontmatter keys like `id`, `country`, `status` — that breaks deduplication.
